Отчёт по лабораторной работе №1 по ООП.

Описание действий, сделанных мной до первого commit.
1.Из проверенного источника была скачана система контроля версий git.
2.Далее она была настроена при помощи данных команд:
git config —global user.name "Your Name"
git config —global user.email "your_email@whatever.com"
git config —global core.autocrlf true
git config —global core.safecrlf warn
git config —global core.quotepath off
3.Далее был создал каталог с название OOP, в котором и будет происходить дальнейшее выполнение лабораторной работы.
4.Далее была произведена инициализация Git в данной папке при помощи пары команд:
cd D:\ООП - Переход в нужную папку.
git init - Инициализация выбранной папки.
5. Создаём текстовый файл для отчёта при помощи данной команды:
touch BelyanskiiArtem_PI-92_LR1.txt
Далее мы его индексируем:
git add BelyanskiiArtem_PI-92_LR1.txt
6. Создаём первый commit:
git commit -m "File Creation"
7. Создаём комит, в котором расказывается о проделанной работе до данного времени.
8. Создаём файл qw.txt, что бы продемонстрировать игнорирование. При помощи команды echo qw.txt > .gitignore создаю папку .gitignore и помещаем туда файл qw.txt. 
Добавляем файл .gitignore при помощи команды git add .gitignore и создаём commit с описанием выполненого действия.
9. Для того, что бы показать сравнение была использована команда git diff хеш1 хеш2.
10. Что бы показать удаление файла мы создаём файл udali_menya.txt, заносим его в репозитории при помощи команды git add.
Далее, при помощи команды git rm -f udali_menya.txt удаляем его.
11. для того, что бы показать перемещение мы создаём файл poezd.txt и добавляем в наш препозитории при помощи команды git add poezd.txt
Далее создаём новую дерикторию depo при помощи команды mkdir depo.
Потом перемещаем powzd.txt в новую папку depo при помощи команды git mv poezd.txt depo .
12. Для просмотра commit'ов используется команда git log.
13. Изменения могут быть индексированными и неиндексированными. 
Что бы отменить неиндексированные изменения следует воgitспользоваться командой checkout: вносим изменения в предварительно созданный и добавленный в репозиторий файл qawsed.txt.
Вносим в него какие либо изменения и потом отменяем их при помощи команды git checkout qawsed.txt .
Что бы отменить индексированные изменения сперва используем команду git reset HEAD qawsed.txt, а потом уже git checkout qwsed.txt. 
14. Работа с ветками. Создаём новый файл branch.txt и изменяем его в ветке master. Потом создаём вторую ветку и называем её beta при помощи команды git checkout -b "beta".
Редактируем в данной ветке наш файл и создаём новый commit. Далее с пощью команды log можно увидеть что наш файл находится в разных ветках и, более того, имеет разное наполднение.
Для объеденения веток используем команду git merge beta. Далее через команду log мы можем увидеть, что наш файл принял одно значение/
Для удаления ветки воспользуемся командой git branch -D beta.
Самый простой конфликт - когда при слияние веток наш файл принимает одно из значений. Самое простое решение - просто переписать то, что нам нужно в этот файл вручную. 
15. Создаём файл zxc.txt и изменяем его. после того как мы введём команду git status мы увидим, что есть индетифицированные изменения. Вводим команду git stash, а потом опять git status - изменения в данном файле пропали.
После вводим команду git stash pop и после ввода git status нам бует видно, что данный файл мы как будто только что добавили.
